RadReveal

Reveal.js add-ons

A JavaScript library to help you extend
Reveal.js - the HTML Presentation Framework.

Reveal.js rocks!

  • Web Developers love Reveal.js.
  • I love Reveal.js because it is hackable.

...but

  • But not a lot of people customize or extend it.
  • Or if they are, they aren't sharing!

The solution? RadReveal

  • Makes customizing Reveal.js easy.
  • Attach behaviors to slides using data attributes.
  • Encourages small, shareable packages.

Simple example

What does an add-on look like?

rad-example.js 
var RadReveal = require('rad-reveal');
RadReveal.register(
  'example', //name of the add on
  function(radConfig, slides) { //initialization function
    for(var s = 0, len = slides.length; s < len; s++) {
      slides[s].className += ' ' + radConfig.addClass;
    }
  });

How is it loaded?

Reveal.initialize({
  ... normal Reveal configuration here
  
  dependencies: [
    { 
      //include it just like a normal dependency, 
      //  but with radName and radConfig properties
      src: 'somepath/rad-example.js', 
      radName: 'example',
      radConfig: { addClass: 'example' } 
    }
    ... other dependencies go here
  ]
});

require('rad-require').initialize();

Simple example

How does that work?

  1. Reveal.initialize loads dependencies, including rad-example.js.
  2. Example add-on registers self with require('rad-require').register({ ... }).
  3. After Reveal is initialized, we call require('rad-require').initialize();.
  4. RadReveal's initialize calls example add-on's initialize, passing radConfig, allSlides
  5. Example add-on does whatever it wants.
  6. In this example, the add-on appends a class to every slide.

Beyond initialize

Imagine you want to run some code against specific slides at certain points in the slideshow.

How could you do that?

Beyond initialize

Reveal.js provides some data attributes you can attach to slides to cause certain behaviors (different transitions, backgrounds, etc).

RadReveal provides a similar approach.

Beyond initialize

You can provide RadReveal attributeEventListeners to define an attribute listener function for your add-on.

  • Listener functions are only called for slides or fragments that have specific attributes.
  • and only when specific events occur

Beyond initialize

For example:

require('rad-require').register({
  name: 'example',
  attributeEventListeners: {
    'data-rad-example': {
      setup: someFunction
    }
  }
});

For any slide with the data-rad-example attribute, someFunction will be called as part of add-on setup.

Beyond initialize

These events can be used for attributeEventListeners, called for every slide or fragment with the specified attribute:

  • setup is called when the add-on is initialized.
  • shown is called when the matching slide is displayed.
  • hidden is called when the matching slide is hidden.
  • fragmentSetup is called when the add-on is initialized.
  • fragmentShown is called when the fragment is displayed.
  • fragmentHidden is called when the fragment is hidden.

Beyond initialize

The listener function is called with four arguments:

  • attrVal, the value of the attribute for this slide.
  • slideObj, an object representing the slide.
  • event, the event object.
  • radEventName, the name of the event.

Beyond initialize

So you can use this to do something to the slide element. For example, we could add a class based on the value of the attribute.

function addTheClass(attrVal, slideObj, event, radEventName) {
  slideObj.element.className += ' ' + attrVal;
}

require('rad-require').register({
  name: 'example',
  attributeEventListeners: {
    'data-rad-example': {
      setup: addTheClass
    }
  }
});

When arriving at a slide

If you wanted to run a function upon showing or hiding a slide with a specific attribute, you could do this:

require('rad-require').register({
  name: 'example',
  attributeEventListeners: {
    'data-rad-example': {
      shown: anotherFunction,
      hidden: andAnotherFunction
    }
  }
});

The functionRunner
add-on.

functionRunner is a simple add-on that triggers a function to run for any slide whose section tag has a
data-rad-functionrunner-{eventName} attribute.

Run something on setup

This slide has a data-rad-functionrunner attr:

<section data-rad-functionrunner-load='{"root":"bar","func":"baz"}'>

and the page has a (global) bar object defined:

var bar = {
  value: 'Hello world!',
  baz: function(slideObj, event, radEventName) {
    var span = document.createElement('span');
    span.innerHTML = radEventName + ': ' + this.value;
    slideObj.element.appendChild(span);
  }
}

which together results in the text being added to the slide:

Run when slide is shown

This slide has a data-rad-functionrunner-shown attr:

<section 
  data-rad-functionrunner-shown='{"func":"foo","args":["shown","2"]}'>

and the page has a (global) function foo defined:

function foo(arg0, arg1, slideObj, event, radEventName) {
  var span = document.createElement('span');
  span.innerHTML =  radEventName + ': ' + arg0 + ' at ' + arg1 + '; ';
  slideObj.element.appendChild(span);
}
